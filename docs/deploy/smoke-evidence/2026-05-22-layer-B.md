### Phase 4 で発覚した latent issues (本 PR fix 対象、plan §1.3 4 件以外の追加 5-6-7 件目)

| # | Issue | 根本原因 | Fix |
|---|---|---|---|
| 5 | api container 起動失敗 (`ModuleNotFoundError: No module named 'scripts'`) | `Dockerfile.api` に `COPY scripts ./scripts` 欠落 (`backend/app/db/active_registry_mutation_gate.py:26` + 他 2 箇所で `from scripts import taskhub_active_registry_gate` するが scripts module が image 内に無い) | `Dockerfile.api` に `COPY --chown=appuser:appuser scripts ./scripts` 追加 (本 PR fix) |
| 5b | worker container も同 issue | 同上 (`backend/app/workers/active_registry_worker_gate.py:24`) | `Dockerfile.worker` に同 COPY 追加 |
| 6 | host から container への port publish 不可 (`docker port` 空、`lsof` で host listen なし、`curl http://127.0.0.1:8000/healthz` connection refused) | `docker-compose.yml` で `networks.taskmanagedai_internal.internal: true` 設定 (SP-012 batch 6 `003e4b4` で deny-by-default 強化として導入) が development mode でも適用される。`ports: ["127.0.0.1:8000:8000"]` 設定が ignored | `docker-compose.dev.yml` に `networks.taskmanagedai_internal.internal: false` override 追加 (development mode のみ host access 許可、production の Tailscale 閉域 deny-by-default 設計は不変) |
| 7 | smoke SOP の `docker compose --env-file .env.local up -d` で dev override が auto-detect されない (`docker-compose.override.yml` だけ auto-detect、`.dev.yml` は明示 `-f` 必須) | SOP §2-§5 の全 docker compose command が dev override を明示していない | SOP の全 docker compose command を `docker compose -f docker-compose.yml -f docker-compose.dev.yml --env-file .env.local` に置換 + §0.2 dev override 適用の注記追加 |
| - | dev override の frontend service で `pnpm dev` mode が named volume `frontend_node_modules` 空のため起動失敗 (`sh: next: not found` 反復 restart) | dev override で frontend を hot reload mode にしていたが、smoke 用には不要 (production build が動作確認に十分) | dev override から frontend service section 全体を削除 (smoke / dev 共通で base compose の production frontend を使う、developer の frontend hot reload は host 直接 `cd frontend && pnpm dev` で実行) |
