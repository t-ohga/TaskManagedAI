### 1.4 中核安全境界 (初期から保持)

- **AI 出力直結禁止**: artifact → schema_validated → policy_linted → diff_ready → approval → runner/repo の pipeline。
- **Provider Compliance Matrix**: `payload_data_class ≤ allowed_data_class` を機械判定、未登録/越境は送信前 deny。
- **SecretBroker**: `secret_ref` + 短命 capability token + atomic claim、raw secret は DB/AI/runner/artifact/audit に出さない。
- **AgentRun 16 状態機械** + ContextSnapshot 10 カラム。
- **tenant / project 境界**: 全主要 table に `tenant_id`、複合 FK で越境禁止。
- **Hard Gates 7** (AC-HARD-01〜07) + **Quality KPIs 5** (AC-KPI-01〜05) を eval harness で fixture 化。

### 1.5 運用 CLI (taskhub)

`scripts/taskhub_*.py`: backup (pg_dump + Redis BGSAVE + artifacts tar + age 暗号化 + checksums) / restore orchestrator / approval CLI / KPI baseline / keyring / remote status 等。

## 2. Mac 運用開始 手順 (最短経路)

> 正本は [`mac-single-host-smoke-sop.md`](./mac-single-host-smoke-sop.md)。ここは要点の最短版。
> 個人 dev 運用は **dev override** (`-f docker-compose.dev.yml`) を必ず付ける (HTTP loopback 動作)。

### 前提 (一度だけ)
```bash
# Docker Desktop 起動中 / port 5432・6379・8000・3900 空き / disk ≥ 5GB
brew install age jq
```

### Step 0. 最新化 (最優先)
```bash
cd ~/repo/TaskManagedAI
git fetch --prune origin
git status -sb                       # 現在ブランチと ahead/behind を確認
# main 運用なら main へ切替えて fast-forward 取り込み (feature ブランチに main を誤 merge しない)
git switch main && git pull --ff-only origin main
```

### Step 1. .env.local 設定
```bash
# 既に .env.local がある場合は中身確認のみ。新規なら:
cp .env.example .env.local
cat >> .env.local <<'ENV'
TASKMANAGEDAI_ENVIRONMENT=development
POSTGRES_PASSWORD=<32字以上の任意値>
TASKMANAGEDAI_DEV_LOGIN_COOKIE_SECRET=<32字以上の任意値>
TASKMANAGEDAI_DEV_LOGIN_TOKEN=<任意値>
TASKMANAGEDAI_DATABASE_URL=postgresql+asyncpg://taskmanagedai:<上のPW>@postgres:5432/taskmanagedai
TASKMANAGEDAI_REDIS_URL=redis://redis:6379/0
ENV
```

### Step 2. (AI 実行を使う場合のみ) Provider API キー
- OpenAI / Anthropic / Gemini キーを SOPS+age で `config/secrets/` に登録 (`.sops.yaml` の age 公開鍵で暗号化)。
- **使わない場合は不要** — Mock Provider Adapter で全機能の動作確認が可能。

### Step 3. build → up → migrate → seed
```bash
DC="docker compose -f docker-compose.yml -f docker-compose.dev.yml --env-file .env.local"
$DC build                                   # 初回 5-15 min
$DC up -d                                    # 5 service が healthy になるまで待機
bash scripts/alembic_wrapper.sh upgrade head # migration 適用 (pipefail で失敗検知)
$DC exec -T api uv run --no-sync python -m backend.app.seeds.runner  # default actor seed (これがないと login が 401)
```

### Step 4. アクセス
```
http://127.0.0.1:3900   →  dev login  →  /today
```

### Step 5. バックアップ drill (運用前に一度)
```bash
# taskhub backup で pg_dump + Redis BGSAVE + artifacts tar + age 暗号化 + checksums 整合を確認
```
