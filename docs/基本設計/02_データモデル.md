# データモデル

## 1. 目的

本書は TaskManagedAI P0 のデータモデル原則、bounded context、主要テーブル、tenant 境界 invariant、ContextSnapshot、AgentRun event、Claim / Evidence 正規化、越境 negative test、RLS policy 草案を定義する。

P0 は個人 1 user で運用するが、商用化準備として schema は最初から tenant-aware にする。

## 2. データモデル原則

| 原則 | 内容 |
|---|---|
| tenant invariant | 全主要テーブルに `tenant_id bigint NOT NULL DEFAULT 1` を持たせる |
| 複合 FK | 親子参照は `tenant_id` を含む複合 FK で閉じる |
| 一意制約 | slug、key、seq などは原則 `tenant_id` を含める |
| actor / principal 分離 | human、service、agent、provider、github_app を同じ監査主体として扱う |
| secret_ref 抽象化 | secret 値は DB に保存せず `secret_ref` URI のみ保存する |
| append-only event | AgentRunEvent、AuditEvent、PolicyDecision は追記を正本にする |
| artifact 経由 | AI 出力は artifact として保存し、採否判定後に domain へ反映する |
| rls_ready metadata | P0 は RLS 無効のまま、`rls_ready: true` と policy 草案を残す |
| app_role | P0 の app は owner ではなく `app_role` からアクセスする |
| negative test | SELECT / INSERT / UPDATE / DELETE 越境を DB 制約 + app_role で失敗させる |

## 3. Bounded Context

| Context | 主なテーブル |
|---|---|
| Task Core | tenants、users、actors、principals、workspaces、projects、repositories、tickets、ticket_relations、acceptance_criteria |
| Agent Runtime | agent_runs、agent_run_events、artifacts、context_snapshots、budgets |
| Research | research_tasks、claims、evidence_sources、evidence_items |
| Policy | policy_rules、approval_requests、policy_decisions |
| Tool | tool_registry、tool_versions、tool_permissions |
| Eval | eval_runs、eval_cases、eval_scores、dataset_versions |
| Audit | audit_events、notification_events |

## 4. テーブル一覧と主要カラム

以下は P0 基本設計の DDL 抜粋である。実 migration では型、制約名、timestamp trigger、comment、migration role を Sprint Pack で確定する。

### 4.1 Tenant / Actor Core

```sql
create table tenants (
  id bigint primary key,
  slug text not null,
  display_name text not null,
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,
  created_at timestamptz not null default now(),
  unique (slug)
);

create table users (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  email text null,
  display_name text not null,
  dev_login_subject text null,
  status text not null check (status in ('active','disabled')),
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,
  created_at timestamptz not null default now(),
  unique (tenant_id, id),
  unique (tenant_id, dev_login_subject)
);

create table actors (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  actor_type text not null check (
    actor_type in ('human','service','agent','provider','github_app')
  ),
  actor_id text not null,
  user_id uuid null,
  impersonated_by uuid null,
  auth_context_hash text not null,
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,
  created_at timestamptz not null default now(),
  unique (tenant_id, id),
  unique (tenant_id, actor_id),
  foreign key (tenant_id, user_id) references users(tenant_id, id),
  foreign key (tenant_id, impersonated_by) references actors(tenant_id, id)
);

create table principals (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  actor_id uuid not null,
  principal_type text not null check (
    principal_type in ('session','api_token','capability_token','installation','worker')
  ),
  subject text not null,
  expires_at timestamptz null,
  auth_context_hash text not null,
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,
  created_at timestamptz not null default now(),
  unique (tenant_id, id),
  foreign key (tenant_id, actor_id) references actors(tenant_id, id)
);
```

create table agent_runs (

### 4.6 ContextSnapshot

```sql
create table context_snapshots (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  run_id uuid not null,
  snapshot_kind text not null check (
    snapshot_kind in ('input','pre_tool','post_tool','resume','final')
  ),
  prompt_pack_version text not null,
  prompt_pack_lock jsonb not null,
  policy_version text not null,
  policy_pack_lock jsonb not null,
  repo_state jsonb not null,
  tool_manifest jsonb not null,
  evidence_set_hash text null,
  provider_continuation_ref jsonb null,
  provider_request_fingerprint jsonb not null,
  state_json jsonb not null default '{}'::jsonb,
  exportable boolean not null default true,
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,
  created_at timestamptz not null default now(),
  unique (tenant_id, id),
  unique (tenant_id, run_id, id),  -- agent_runs.current_snapshot_id 参照用の複合 unique
  foreign key (tenant_id, run_id) references agent_runs(tenant_id, id)
);

-- agent_runs.current_snapshot_id は循環参照のため deferrable で後置定義
-- 別 run の snapshot を current として保存できないことを DB で保証する
alter table agent_runs
  add constraint agent_runs_current_snapshot_fk
  foreign key (tenant_id, id, current_snapshot_id)
  references context_snapshots(tenant_id, run_id, id)
  deferrable initially deferred;
```

ContextSnapshot の必須 10 カラムは次で固定する。

1. `prompt_pack_version`
2. `prompt_pack_lock`
3. `policy_version`
4. `policy_pack_lock`
5. `repo_state`
6. `tool_manifest`
7. `evidence_set_hash`
8. `provider_continuation_ref`
9. `provider_request_fingerprint`
10. `snapshot_kind`

### 4.8 Tool / Eval / Audit

```sql
-- P0 migration: P0 制約を deny-by-default で DB レベルに固定
-- transport は local/stdio のみ、auth_mode は none/env_ref のみ、network_access=false 強制
-- remote_http / oauth / capability_token は P1 migration で別途解禁する
create table tool_registry (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  tool_key text not null,
  transport text not null check (transport in ('local','stdio')),
  auth_mode text not null check (auth_mode in ('none','env_ref')),
  network_access boolean not null default false check (network_access = false),
  trust_tier text not null check (
    trust_tier in ('official','self_hosted','third_party','experimental')
  ),
  manifest jsonb not null,
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,
  created_at timestamptz not null default now(),
  unique (tenant_id, id),
  unique (tenant_id, tool_key)
);

create table tool_versions (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),

  foreign key (tenant_id, tool_id) references tool_registry(tenant_id, id)

create table tool_permissions (

create table dataset_versions (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  dataset_key text not null,
  version text not null,
  fixture_kind text not null check (
    fixture_kind in ('public_regression','private_holdout','adversarial_new')
  ),
  content_hash text not null,

  -- / adversarial_new) を持ち得るため、fixture_kind を unique key に含める (Sprint 11 batch 5a PR #28

  unique (tenant_id, dataset_key, version, fixture_kind)
);

create table eval_runs (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  run_id uuid null,
  dataset_version_id uuid not null,
  suite_name text not null,
  provider text not null,
  model text not null,
  summary jsonb not null default '{}'::jsonb,
  started_at timestamptz not null default now(),
  ended_at timestamptz null,

  foreign key (tenant_id, dataset_version_id) references dataset_versions(tenant_id, id)
);

create table eval_cases (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  dataset_version_id uuid not null,
  case_key text not null,
  case_json jsonb not null,
  expected_json jsonb not null,
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,

-- ============================================================
-- Secrets registry (DD-06 SecretBroker の永続化裏側)
-- raw secret は保存しない。backend (local: OS keychain/暗号化file Phase 0 / sops: SOPS+age D-4、ADR-00058) に保管し、ここには参照のみ
-- ============================================================
create table secret_refs (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  secret_uri text not null,                       -- 形式: secret://<backend>/<scope>/<name>#<version> (backend=local|sops、ADR-00058、Phase 0 default=local、SECRET_URI_PATTERN 集約)
  scope text not null check (
    scope in ('p0','workspace','project','repo','agent_run','provider')
  ),
  name text not null,
  version text not null,
  -- DD-06 rotation 手順との整合: pending(rotation 中の新 version) / active(現行) / deprecated(置き換え後の旧 version) / revoked(緊急失効)
  status text not null check (status in ('pending','active','deprecated','revoked')),
  -- ADR-00058/00059 material lifecycle (create/rotate/revoke crash-safe source of truth、backend material は tenant_id+secret_ref_id 束縛で store 側保管)
  material_state text not null default 'present' check (material_state in ('writing','present','purging','purged')),
  material_purged_at timestamptz null,            -- revoke 後 material purge 完了時刻 (NULL=未 purge、revoked AND NULL = 要 gc-orphans)
  purge_attempts integer not null default 0,      -- material purge 再試行回数 (reconciliation)
  runner_injectable boolean not null default false check (runner_injectable = false),  -- 常に false を強制
  allowed_consumers jsonb not null,               -- caller allowlist: ["api:repo_proxy", "api:provider_adapter"] 等
  allowed_operations jsonb not null default '[]'::jsonb,  -- operation allowlist: ["git_push", "provider_call.openai_responses"] 等
  owner_actor_id uuid not null,
  rotated_from_id uuid null,                      -- 前 version を指す（rotation 履歴）
  metadata jsonb not null default '{"rls_ready": true}'::jsonb,
  created_at timestamptz not null default now(),
  deprecated_at timestamptz null,                 -- deprecated 化日時（rotation 完了時刻）
  revoked_at timestamptz null,                    -- 緊急失効日時
  unique (tenant_id, id),
  unique (tenant_id, secret_uri),
  unique (tenant_id, scope, name, version),
  foreign key (tenant_id, owner_actor_id) references actors(tenant_id, id),
  foreign key (tenant_id, rotated_from_id) references secret_refs(tenant_id, id)
);

-- 各 (tenant_id, scope, name) で active は最大 1 つ、pending は最大 1 つ
create unique index secret_refs_one_active_per_name
  on secret_refs (tenant_id, scope, name) where status = 'active';
create unique index secret_refs_one_pending_per_name
  on secret_refs (tenant_id, scope, name) where status = 'pending';

-- SecretBroker issue invariant (実装 contract): token 発行時に
-- 1. requested_operation ∈ secret_refs.allowed_operations
-- 2. caller ∈ secret_refs.allowed_consumers
-- 3. 通常 operation: status = 'active' のみ発行可
-- 4. rotation.verify 専用 operation: status = 'pending' でも発行可（caller / operation を別扱い）
-- 5. deprecated / revoked からの発行は禁止
-- を必須チェック。違反は token 発行前に deny audit へ落とす。

-- SecretBroker redeem invariant (実装 contract): atomic claim UPDATE で
-- 1. token_hash 一致
-- 2. status = 'issued', used_at IS NULL, expires_at > now()
-- 3. issued_to_actor_id = redeem 時の actor_id (bearer 盗難対策)
-- 4. issued_run_id IS NOT DISTINCT FROM redeem 時の run_id
-- 5. expected_request_fingerprint = :computed_fingerprint
--    (broker が redeem 時の実 operation request から OperationContext canonical schema で再計算した値。
--     caller が任意 hash を渡せない設計。issue 時に束縛、redeem 時に照合する。)
-- 6. requested_operation ∈ allowed_operations
-- を 1 文で検証 + status='redeeming' に遷移。0 件 RETURNING は deny。
-- redeem 後は同一 transaction 内で secret_refs を for update し、status / allowed_consumers /
-- allowed_operations / scope の整合を再検証。不一致なら raw secret resolve せず deny。

-- ============================================================
-- Capability token lifecycle (短命 TTL 5-30 分、redeem 1 回限り)
-- raw token は保存せず、token_hash のみ保管
-- ============================================================
create table secret_capability_tokens (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  secret_ref_id uuid not null,
  token_hash text not null,                       -- raw token は保存しない（SHA-256 等のハッシュ）
  allowed_operations jsonb not null,              -- ["git_push", "provider_call.openai_responses"] 等
  scope_constraint jsonb not null default '{}'::jsonb,  -- 操作の追加制約（repo / provider / 等）
  issued_to_actor_id uuid not null,
  issued_run_id uuid null,
  expires_at timestamptz not null,
  used_at timestamptz null,                       -- redeem 時刻（atomic claim 時に確定）
  expected_request_fingerprint text not null,     -- issue 時に broker が canonical OperationContext から計算
                                                  -- redeem 時の :computed_fingerprint と一致しないと deny
                                                  -- caller supplied fingerprint は禁止（設計違反として reject）
  -- DD-06 atomic claim 設計: issued → redeeming → used の遷移を一発 UPDATE で行う
  -- 'redeeming' は claim 後 / operation 完了前の状態（broker クラッシュ時は expires_at で自動 expire）
  status text not null check (status in ('issued','redeeming','used','expired','revoked')),
  metadata jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  unique (tenant_id, id),
  unique (tenant_id, token_hash),
  foreign key (tenant_id, secret_ref_id) references secret_refs(tenant_id, id),
  foreign key (tenant_id, issued_to_actor_id) references actors(tenant_id, id),
  foreign key (tenant_id, issued_run_id) references agent_runs(tenant_id, id)
);

-- 検索性: 失効間近 / 未使用 token を監視
create index secret_capability_tokens_expires_idx
  on secret_capability_tokens (tenant_id, expires_at)
  where status = 'issued';
```

## 5. 複合 FK 例

tickets は `project_id` だけで projects を参照しない。必ず `tenant_id` と組み合わせる。

```sql
create table tickets (
  id uuid primary key,
  tenant_id bigint not null default 1 references tenants(id),
  project_id uuid not null,
  title text not null,
  unique (tenant_id, id),
  foreign key (tenant_id, project_id) references projects(tenant_id, id)
);
```

この制約により、別 tenant の project_id を持つ ticket を INSERT できない。

### 5.1 project 境界 invariant（同一 tenant 内、project 内）

tenant 境界に加え、**同一 tenant 内で project が異なる resource をクロス参照させない**ことも DB レベルで防ぐ。tickets / research_tasks / agent_runs / repositories の親子・関連参照を `(tenant_id, project_id, id)` の複合 FK で閉じる。

```sql
-- 親側に project_id を含む複合 unique を追加
alter table repositories add constraint repositories_uq_tenant_project_id
  unique (tenant_id, project_id, id);

alter table tickets add constraint tickets_uq_tenant_project_id
  unique (tenant_id, project_id, id);

alter table research_tasks add constraint research_tasks_uq_tenant_project_id
  unique (tenant_id, project_id, id);

-- 子側: tenant + project + parent id の複合 FK
alter table tickets add constraint tickets_repository_in_same_project
  foreign key (tenant_id, project_id, repository_id)
  references repositories(tenant_id, project_id, id);

alter table research_tasks add constraint research_tasks_ticket_in_same_project
  foreign key (tenant_id, project_id, ticket_id)
  references tickets(tenant_id, project_id, id);

alter table agent_runs add constraint agent_runs_ticket_in_same_project
  foreign key (tenant_id, project_id, ticket_id)
  references tickets(tenant_id, project_id, id);

alter table agent_runs add constraint agent_runs_research_task_in_same_project
  foreign key (tenant_id, project_id, research_task_id)
  references research_tasks(tenant_id, project_id, id);

-- agent_runs の lineage（parent_run_id）も同一 project 内に閉じる
alter table agent_runs add constraint agent_runs_uq_tenant_project_id
  unique (tenant_id, project_id, id);

alter table agent_runs add constraint agent_runs_parent_run_in_same_project
  foreign key (tenant_id, project_id, parent_run_id)
  references agent_runs(tenant_id, project_id, id);

-- ticket_relations も project 境界に閉じる（cross-project relation は別モデルで分離）
alter table ticket_relations add column project_id uuid not null;

alter table ticket_relations add constraint ticket_relations_project_fk
  foreign key (tenant_id, project_id)
  references projects(tenant_id, id);

alter table ticket_relations add constraint ticket_relations_source_in_same_project
  foreign key (tenant_id, project_id, source_ticket_id)
  references tickets(tenant_id, project_id, id);

alter table ticket_relations add constraint ticket_relations_target_in_same_project
  foreign key (tenant_id, project_id, target_ticket_id)
  references tickets(tenant_id, project_id, id);

-- nullable FK は MATCH SIMPLE のため、ticket_id / research_task_id / parent_run_id が null の場合は許容される
```

越境 negative test fixture には**同一 tenant・別 project**ケース（特に `ticket_relations` で別 project の ticket を結ぶ INSERT、`agent_runs.parent_run_id` を別 project の run に向ける INSERT/UPDATE）も含める。

## 7. 越境 negative test fixture の方針

P0 の Hard Gate `tenant_isolation_negative_pass` は次を fixture 化する。

| 操作 | 期待結果 |
|---|---|
| SELECT | app_role の tenant context と異なる `tenant_id` の行を取得できない |
| INSERT | 別 tenant の parent id を参照する行を INSERT できない |
| UPDATE | tenant context 外の行を UPDATE できない |
| DELETE | tenant context 外の行を DELETE できない |
| INSERT (同一 tenant・別 project) | 別 project の parent id を参照する行を INSERT できない（複合 FK で失敗） |
| UPDATE (同一 tenant・別 project) | resource を別 project の parent へ付け替える UPDATE が失敗する |

P0 では RLS を有効化しないが、app repository layer は常に `tenant_id = current_tenant_id` を WHERE に入れる。INSERT は複合 FK で失敗させる。UPDATE / DELETE は app_role の repository contract test で対象行数 0 を確認する。

## 8. evidence_set_hash 正規化アルゴリズム

`evidence_set_hash` は ContextSnapshot の再現性の核である。

計算手順は次で固定する。

1. Claim text、EvidenceSource、EvidenceItem の reader-facing text を NFC UTF-8 に正規化する。
2. URL は scheme / host の正規化、不要 fragment の扱い、末尾 slash の扱いをルール化し、`canonical_url` に統一する。
3. JSON は JCS canonical JSON で canonicalize する。
4. claim は `claim_id` 昇順、source は `source_id` 昇順、evidence item は `claim_id`、`source_id`、`relation` 昇順に並べる。
5. `provenance_json` は PROV-DM 互換 bundle として canonicalize する。
6. 正規化 bundle 全体の SHA-256 を `evidence_set_hash` として保存する。

P0 ではこの hash を ContextSnapshot と Eval に保存し、citation coverage と再現性の検証に使う。

## 9. provider_request_fingerprint の保存内容

`provider_request_fingerprint` は provider call の再現性と model alias 変更検知のために必須である。

保存内容は次で固定する。

```json
{
  "provider": "openai",
  "model_requested": "string",
  "model_resolved": "string",
  "api_version": "string",
  "sdk_version": "string",
  "region": "string",
  "temperature": 0,
  "top_p": 1,
  "max_tokens": 4096,
  "safety_settings": {},
  "tool_schema_hash": "sha256",
  "response_format_schema_hash": "sha256"
}
```

P0 の DDL では JSONB として保存し、Sprint 5 の ProviderAdapter contract test で必須キーの存在を検証する。

## 10. RLS policy 草案

P0 では RLS は有効化しない。P1 で有効化するため、各テーブルに `tenant_id` と `metadata.rls_ready=true` を持たせる。

P1 の policy 草案は次である。

```sql
alter table tickets enable row level security;

create policy tenant_select_tickets on tickets
  for select
  using (tenant_id = current_setting('app.tenant_id')::bigint);

create policy tenant_insert_tickets on tickets
  for insert
  with check (tenant_id = current_setting('app.tenant_id')::bigint);

create policy tenant_update_tickets on tickets
  for update
  using (tenant_id = current_setting('app.tenant_id')::bigint)
  with check (tenant_id = current_setting('app.tenant_id')::bigint);

create policy tenant_delete_tickets on tickets
  for delete
  using (tenant_id = current_setting('app.tenant_id')::bigint);
```

P0 の acceptance では、RLS を有効化していない状態でも、app_role、repository layer、複合 FK、negative test により越境が失敗することを確認する。
