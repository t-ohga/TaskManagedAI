# 03 ゲートと非採用項目

作成日: 2026-05-14

## 1. Hard Stop Gate

以下は、修正案や自律化の議論であっても緩めません。

| 条件 | 判定 | 必須 gate |
|---|---|---|
| 破壊的 shell、`git reset --hard`、`git clean`、広範囲 chmod/chown | BLOCK | 明示的ユーザー承認、diff、rollback |
| DB schema / migration / production data 操作 | BLOCK | ADR Gate、migration plan、rollback、tests |
| auth/authz / policy / approval / SecretBroker / provider / runner / MCP write boundary 変更 | BLOCK | security gate、negative tests |
| provider call を PolicyDecision / BudgetGuard / Provider Compliance / audited gateway なしに送る | BLOCK | preflight + enforce + audit |
| AI / agent / service / provider が approval decider になる | BLOCK | human-only service/DB/test gate |
| Runner/MCP mutation gateway 完成前の command/file mutation 自律化 | BLOCK | Runner Boundary Gate、forbidden path/dangerous command tests |
| external publish / deploy / public exposure | BLOCK | Network boundary ADR、user approval |
| missing Sprint Pack / ADR ref のまま Sprint 実装着手 | BLOCK | canonical registry |
| unresolved Quality Loop finding / harness incident のまま fixed | BLOCK | review resolution、clean evidence、validator |

## 2. Provider Feature Gate

Provider feature は provider 名単位ではなく feature 単位にします。

| Feature | P0判断 | Resume condition |
|---|---|---|
| `openai/responses` structured supervisor | 条件付き採用 | `store=false`、schema contract、Provider Matrix、BudgetGuard、OutputValidator |
| `openai/realtime_calls_unified` | defer | actor/run/origin binding、SecretBroker、sideband、retention、budget、modality cap |
| `openai/realtime_client_secrets` | defer secondary | short-lived secret、raw value no-store、browser config deny、origin/CSRF binding |
| `openai/realtime_session_audio` | defer | audio data class、consent、retention、ZDR/MAM、pricing、delete path |
| `openai/realtime_sideband` | defer | server-only tool/session update、audit、forbidden browser event deny |
| `openai/realtime_mcp_direct` | reject product | Tool/MCP Gateway が完全に仲介できるまで product 不可 |
| `gemini/generate_content` narrow structured output | 条件付き採用 | public-only Matrix entry、SP-005 adapter、mock parity、schema validation |
| Gemini managed grounding/search | defer | feature Matrix、source snapshot、citation verifier、retention/region |
| Gemini Code Execution / Computer Use | reject direct | Runner/Tool Gateway、sandbox、artifact、approval、audit が揃うまで不可 |
| Gemini URL Context | defer | URL allow/deny、snapshot/hash、trust classification、network policy |
| Gemini Batch Prediction | defer | async job ledger、external output deletion、cost cap、data residency |
| Gemini Live API | defer high-risk | realtime ADR、audio/video retention、short-lived token、tool-call mixing gate |
| Gemini Model Optimizer | defer | `model_resolved`、router decision、per-model compliance、eval bake-off |

## 3. Rejected Patterns

| Pattern | Reject reason |
|---|---|
| regex JSON tool-call extraction | 壊れやすく、security boundary にならない |
| unrestricted `/api/responses` proxy | client が model/schema/store/tool/budget/payload class を支配できる |
| browser-side mutating tool execution | untrusted client が policy / approval を迂回する |
| raw request/response logging | secret / PII / confidential leakage |
| audio recording on by default | consent、retention、download permission、deletion が未定義 |
| handoff as authorization | role は dispatch hint であり capability ではない |
| guardrail fail-open | Output Validator の fail-closed 方針と衝突 |
| provider grounding as truth | TaskManagedAI evidence snapshot と citation verifier が必要 |
| provider cache as ContextSnapshot source of truth | provider state は optimization metadata に限定 |
| model optimizer as hidden routing | resolved model / compliance / cost / eval trace がない |
| generated benchmark as gold answer | human validation 前は seed に限る |

## 4. Defer Items

| Item | Defer理由 | 再開条件 |
|---|---|---|
| Realtime task intake | Provider Matrix、SecretBroker、sideband、retention、budget が未完成 | text-only / STT chained 比較で明確に勝つ |
| Realtime review session | voice consent と read-only review UI が未定義 | explicit UI approval と分離できる |
| Realtime tracing | data residency / ZDR / trace retention が未確認 | separate Matrix row と default disabled |
| image/screen realtime input | privacy / modality cap 未定義 | separate privacy review |
| managed URL context | snapshot/hash/trust/citation verifier なし | URL policy と evidence capture |
| batch prediction | external storage と deletion/data residency 未設計 | async job ledger と retention policy |
| memory bank / always-on memory | retention / deletion / topic governance 未設計 | `memory_records` と audit policy |
| custom embeddings provider direct use | model/dimension/task_type/re-embedding 未設計 | embedding metadata schema と gold set |

## 5. Auto-Allow と Approval の境界

`low_risk_auto_allow` は approval ではありません。

許可できる形:

- Policy Engine が effect=`allow` を返す。
- 対象 action_class と allowed_actions が profile に明示される。
- budget、max duration、max retries、max children、kill switch がある。
- audit event と policy decision が残る。
- UI/CLI に auto-allowed reason が出る。

禁止する形:

- AI agent が `decided_by_actor_id` になる。
- orchestrator が approval decider になる。
- reviewer pass を human approval の代替にする。
- stale diff / stale policy / stale artifact の承認を再利用する。
- low-risk profile で secret_access / merge / deploy を許す。

## 6. Rollback / Kill Switch

新しい runtime / Realtime / provider feature を入れる場合は、最低限次を持つ。

- feature flag disable。
- provider capability token revoke。
- BudgetGuard cap zero。
- sideband / session init route deny。
- Tool/MCP Gateway deny。
- rollback AuditEvent。
- non-retained audio/transcript/tool output の削除確認。
- Provider Matrix row を blocked に戻す手順。

## 7. 実装前チェックリスト

実装へ進む前に確認すること:

- 対象 Sprint Pack / ADR が存在する。
- write scope と rollback が小さく分かれている。
- action_class が7種正本に合う。
- Provider Matrix が feature 単位である。
- BudgetGuard は事後記録ではなく pre-spend gate。
- SecretBroker は raw secret を artifact/log/snapshot/approval/audit に出さない。
- Output Validator は fail-closed。
- Tool output / transcript / browser event / remote MCP output は `untrusted_content`。
- UI/CLI はユーザーの next action を説明できる。
- test / fixture / manual verification がある。
