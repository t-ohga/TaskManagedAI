作成日: 2026-05-14

## 1. このディレクトリ自体の完了判定

`docs/設計検討/修正まとめ/` は、次を満たしたときに統合成果物として扱える。

| 条件 | 判定方法 |
|---|---|
| README がある | `README.md` が読み順と扱いを示す |
| 統合方針がある | `00_統合修正方針.md` が採用/延期/却下と invariant を示す |
| Roadmap がある | `01_P0優先修正ロードマップ.md` が順序、対象、stop gate を示す |
| 領域別改善案がある | `02_領域別改善案.md` が Product/UI/QL/Autonomy/CLI/Provider/Evidence/Realtime/Gemini を含む |
| Gate/defer/reject matrix がある | `03_ゲートと非採用項目.md` が high-risk feature を並べる |
| Source traceability がある | `04_出典トレースマトリクス.md` が入力26ファイルの反映先を示す |
| Upstream destination がある | `05_次に編集する正本ファイル案.md` が次に触る正本を示す |
| Verification criteria がある | 本ファイルが存在する |
| Conflict precedence がある | `07_矛盾と正本優先順位.md` が衝突時の正本を示す |
| Handoff がある | `08_実装担当引き継ぎ.md` が実装担当者向けの読み順、作業分割、禁止範囲、完了条件を示す |

## 3. High-risk gate coverage

次が文書内に明示されていなければ、統合案は不十分。

| Gate | 必須記載 |
|---|---|
| Provider Compliance | provider / feature / payload data class / retention / region / pricing / ZDR/MAM を feature 単位で見る |
| SecretBroker | raw key / ephemeral secret / token response body を artifact/log/UI/audit に残さない |
| Approval | human-only decider、self-approval 禁止、stale invalidation |
| BudgetGuard | provider call 前の estimated max token/cost/wall-clock/retry/repair/kill switch |
| Runner / Tool Gateway | direct command / MCP mutation / provider-managed execution を gateway 外で許可しない |
| Output Validator | schema/policy fail-closed、guardrail fail-open reject |
| Input Trust Layer | transcript/tool output/browser event/remote MCP output は `untrusted_content` |
| Quality Loop | open finding / unresolved harness incident で fixed にできない |
| P0 false-positive | Deep Research-to-PR と daily control plane を通らず P0 Exit にしない |

## 4. Verification before upstream edits

このまとめを使って上流文書を編集する前に、次を確認する。

```bash
find docs/設計検討/修正まとめ -maxdepth 1 -type f | sort
rg -n "P0 false-positive|ResearchTask|Today / Inbox|request_revision|Agent Runs|Quality Loop|ContextResolver|InteractionGateway|Provider Matrix|Gemini Code Execution|Realtime MCP" docs/設計検討/修正まとめ
rg -n "task_write|repo_write|pr_open|secret_access|merge|deploy|provider_call" docs/設計検討/修正まとめ
rg -n "DI-17|project auto-discovery|cross-project memory|deny-by-default" docs/設計検討/修正まとめ
rg -o "A-[A-Z0-9]+-[0-9]{2}" docs/設計検討/修正まとめ/04_出典トレースマトリクス.md | sort -u | wc -l
```

期待:

- 上記キーワードが1ファイルだけでなく、方針、roadmap、領域別、gate、traceability、上流修正案の複数箇所でつながっている。
- `read/search` は policy action_class としてではなく、allowed action / Tool Registry 側に扱われている。
- Realtime / Gemini managed feature は runtime 実装許可として書かれていない。
- Action backlog ID count は 58 以上で、`A-PROD-01` から `A-CLI2-03` までの個別 ID が trace できる。
- 実装担当者が最初に開始すべき run、触ってよい正本候補、触ってはいけない範囲が `08_実装担当引き継ぎ.md` で一意に分かる。
