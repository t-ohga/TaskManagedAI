## 3. SP-012 修正案

追加すべき P0 gold flow:

```text
ResearchTask
-> Claim / Evidence / citation coverage
-> ResearchDecision
-> generated Ticket
-> Plan Loop
-> Approval / Request Revision
-> Implementation / Draft PR
-> Review Loop
-> Eval / Audit / Cost
```

Acceptance:

- generated Ticket 全件が Evidence / Claim / Decision lineage を持つ。
- plan/review/revision/rereview/conformance artifact が残る。
- cost_per_completed_task が再計算できる。
- review finding が open のまま done にならない。
- Research なしの flow は smoke であり、P0 representative gold flow ではない。

## 4. SP-009 修正案

Must ship に追加:

- Dashboard / Today Inbox。
- Approval decision packet。
- request_revision。
- Agent Runs workspace。
- Execution Log summary lane。
- Notification triage queue。
- First-run / beginner workflow。
- minimal KPI strip。

Acceptance:

- Dashboard は system health だけでなく、承認待ち、blocked、failed、ready review、next action を表示する。
- Approval は diff summary、affected files、policy reason、evidence、cost、rollback、stale state を出す。
- request_revision は old approval invalidation と再 validation に接続する。
- Agent Runs は run state、phase、assignment reason、approval state、cost、retryability、child runs を出す。
- raw provider payload / client secret / secret-like value は redacted-by-default。

## 6.1 Project Auto-Discovery / Memory Boundary 修正案

DI-17 は CLI / cross-project safety と同じタイミングで扱う。memory は ContextSnapshot を置き換えず、project/repo/worktree の現在状態を補助する supplement として限定する。

追加:

- project auto-discovery の最小仕様。
- project registry / repo mapping / worktree identity。
- ContextSnapshot と local memory の優先順位。
- cross-project memory retrieval deny-by-default。
- retrieval reason、source project、confidence、redaction state の audit。

Acceptance:

- `tmai context show` が project/repo/worktree/branch/dirty/confidence と memory supplement の有無を表示する。
- mutating command は project/repo boundary が曖昧なら fail-closed。
- memory 由来の提案は source project と freshness を持ち、ContextSnapshot の正本状態を上書きしない。
- project/repo boundary をまたぐ retrieval は明示 allowlist と audit なしでは拒否する。

## 9. ADR-00023 InteractionGateway 修正案

ADR title:

```text
ADR-00023 Interaction Gateway / Realtime Intake
```

Boundary names:

- `InteractionGateway`
- `RealtimeInteractionAdapter`
- `RealtimeSessionBridge`

Scope:

- P0 runtime ではなく P0.1 prototype gate。
- text-only baseline と STT -> ProviderAdapter -> TTS 比較を先に作る。
- Realtime は unified `/v1/realtime/calls` + backend sideband を第一候補。
- browser は media transport と bridge-approved user input のみ。

Provider Matrix candidate rows:

- `openai/realtime_calls_unified_no_content`
- `openai/realtime_client_secrets_no_content`
- `openai/realtime_session_audio`
- `openai/realtime_transcription`
- `openai/realtime_sideband`
- `openai/realtime_tracing`
- `openai/responses_supervisor_text_store_false`
- `openai/responses_guardrail_store_false`

Reject:

- direct Realtime MCP product use。
- browser-supplied `session.update`、`response.tools`、`function_call_output`、`mcp_approval_response`。
- voice "OK" as approval。
