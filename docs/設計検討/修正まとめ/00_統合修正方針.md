# 00 統合修正方針

作成日: 2026-05-14

## 1. 統合対象

今回統合した入力は、次の3系統です。

| 系統 | 入力 | 主な意味 |
|---|---|---|
| Gemini GitHub 取り込み | `docs/設計検討/geminigithubから/` | Gemini / Google Cloud sample から、ProviderAdapter、Tool Registry、RAG/Evidence/Eval、retry、安全 fixture、managed feature gate の知見を抽出したもの |
| OpenAI Realtime Agents 検討 | `docs/設計検討/openairealtimegithubから/` | Realtime UX、InteractionGateway、sideband、voice / text intake 比較、Realtime MCP reject、retention / cost / consent gate を整理したもの |
| 設計問題点・改善点 | `docs/設計検討/設計問題点改善点/` | TaskManagedAI 本体の P0 false-positive、UI/CLI、Quality Loop、autonomy、traceability、runtime gate の不足を洗い出したもの |

## 2. 最上位判断

TaskManagedAI の次の修正方針は、次の順で固定します。

1. P0 Exit が Deep Research を通らずに完了できる状態を止める。
2. Daily control plane と Approval / Agent Runs / Notification / Execution Log を、毎日使える UI として成立させる。
3. Plan / Review / Revision / Rereview / Conformance / Harness Incident を product artifact として保存する。
4. 自律性は approval bypass ではなく、Policy Engine による低リスク auto-allow と説明可能な dispatch で高める。
5. CLI / terminal は `ContextResolver` を先に固定し、曖昧な mutating command を fail-closed にする。
6. Provider / Realtime / Gemini の便利機能は feature 単位 Matrix、ADR、BudgetGuard、SecretBroker、Runner/Tool Gateway、Eval fixture が揃うまで採用しない。

## 3. 外部 sample から採用するもの

Gemini / OpenAI Realtime の sample から採用するのは、runtime や demo code ではなく、次の pattern です。

| 採用 pattern | TaskManagedAI での landing |
|---|---|
| long-running agent の pending signal / resume | `AgentRunEvent`、`pending_signals`、resume reason、state_delta metadata |
| structured output / tool call schema | `ProviderAdapter`、`ToolCall`、`ToolSchema`、`OutputValidator` |
| MCP / tool schema discovery | `Tool Registry`、MCP Gateway、allowlist、schema hash、approval binding |
| grounding / citation trace | `EvidenceItem`、`Claim`、`GroundingSupport`、citation verifier |
| local-first hybrid retrieval / ranking eval | `SearchRun`、`EvidenceSearchHit`、`RankingPolicyVersion`、retrieval gold set |
| bounded retry / timeout / retry budget | Provider-neutral retry policy、BudgetGuard、idempotency key |
| safety ratings / prompt/RAG/tool-output attacks | `SafetyMetadataRef`、secret canary、negative Eval fixtures |
| Chat-Supervisor / sideband | `InteractionGateway`、`RealtimeInteractionAdapter`、structured supervisor |
| transcript + event log UI | AgentRun timeline、redacted debug lane、validation state display |
| text-only / STT chained / Realtime 比較 | `eval/interaction/realtime_intake/` fixture plan |

## 4. 直接採用しないもの

以下は便利でも、TaskManagedAI の安全境界を壊すため直接採用しません。

| 不採用 | 理由 |
|---|---|
| Google ADK / Agent Engine を core runtime にする | Provider / Runner / Approval / Audit を TaskManagedAI が正本に持つ方針と衝突 |
| Vertex Search / managed RAG を P0 source of truth にする | Evidence snapshot、project boundary、Provider Matrix、retention が未確定 |
| Gemini Code Execution / Computer Use direct use | Runner Gateway、sandbox、approval、audit、artifact capture を迂回する |
| URL Context を evidence 正本にする | provider fetch と context injection が混ざり、source snapshot/hash がない |
| Batch Prediction direct use | async job、external storage、retention/deletion、cost/data residency が未設計 |
| Multimodal Live API を P0 に入れる | token、audio/video retention、live tool-call mixing、session audit が未設計 |
| Model Optimizer direct use | resolved model、compliance、eval、cost trace がない |
| Realtime model を ProviderAdapter 置換にする | strict structured artifact contract を満たす根拠が未確認 |
| Realtime MCP direct execution | Tool/MCP Gateway、approval、audit、retention、prompt-injection check を迂回しやすい |
| browser-side business logic / tool handling | untrusted client が policy / tool / approval に触る |
| unrestricted `/api/responses` proxy | schema、model allowlist、budget、auth、Provider Matrix を迂回する |
| raw request/response logging | secret / PII / confidential leakage risk |
| audio recording / download default on | consent、TTL、encryption、download permission、delete path が未定義 |

## 5. TaskManagedAI の不変条件

統合後も、次の条件は緩めません。

- AI output direct execution 禁止。
- action class は `task_write / repo_write / pr_open / secret_access / merge / deploy / provider_call` の7種を正本にする。
- `read/search` は action class ではなく Tool Registry の allowed action / capability に寄せる。
- Approval decider は human-only。agent / orchestrator / service / provider は decider にならない。
- Provider call は `PolicyDecision`、Provider Compliance Matrix、BudgetGuard、audit gateway を通す。
- SecretBroker は raw secret 非露出、one-time claim、operation fingerprint、approval binding を守る。
- Runner / MCP mutation は gateway、sandbox、forbidden path / dangerous command tests が揃うまで BLOCK。
- AgentRun 16 status は勝手に増やさず、substate / blocked_reason / event / artifact で拡張する。
- ContextSnapshot は正本。provider cache、Realtime session、audio、raw transcript は hash/ref/redacted summary に留める。
- harness incident は未解決のまま clean evidence にできない。

## 6. 統合後の最優先修正テーマ

| 優先 | テーマ | 直す理由 |
|---:|---|---|
| 1 | Sprint / Backlog / ADR 正本化 | missing Sprint Pack と `BL-*` / `sprint_pack_ref` drift が残ると、後続修正先を誤る |
| 2 | Deep Research-to-PR P0 gold flow | TaskManagedAI の差別化価値である証拠付き実装管理を P0 Exit に入れる |
| 3 | SP-009 UI heavy 化 | Dashboard が health page のままだと daily tool にならない |
| 4 | Quality Loop artifact 化 | ユーザーが求める「何度もループしてゼロまで」を product behavior として残す |
| 5 | Runtime safety gate 接続 | 自律化前に PolicyDecision / Budget / Provider / Secret / Runner を hard gate にする |
| 6 | CLI ContextResolver | terminal から project/repo/worktree を推測で mutating しない |
| 7 | Evidence / Eval / Retrieval | Research-to-Ticket lineage、citation coverage、retrieval quality を再計算可能にする |
| 8 | InteractionGateway ADR | Realtime は ProviderAdapter ではなく intake boundary として扱う |
