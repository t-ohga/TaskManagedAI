## 1. 業界調査サマリ (4 round Web search、12 reference)

### 1.1 AI Coding Agent / Task Management Tool 主要 pattern

| Tool / Platform | 主要 UI/UX feature | TaskManagedAI 参考点 |
|---|---|---|
| **Linear** | Cmd+K command palette (全 action)、keyboard shortcut (C/X/S/A/P/L)、custom saved views、bulk action、universal search | Ticket / Approval / Run の全 action keyboard 化 |
| **Cursor** | MCP integration (Postgres/GitHub/Sentry/Linear/Slack)、agent reach to systems | Tool Registry (SP-0045) との連携、外部 system MCP |
| **Cline** | Plan / Act mode separation + step approval (VS Code) | AI Plan → Human Approval → Runner execution flow を Cline pattern で強化 |
| **OpenHands (旧 OpenDevin)** | Full PR generation in sandboxed Docker、test suite + diff/PR review、72k stars | RepoProxy (SP-008) + Docker isolated runner (SP-007) と直結 |
| **LangSmith** | Node-by-node state diff、execution graph、replay vs new model、step-level cost attribution、LangGraph Studio v2 time-travel | AgentRun timeline UI、provider replay、cost dimension |
| **Langfuse** | OSS、self-hostable、OTel-native、MIT license | Sprint 11.5 operational_hardening で採用候補 (BL-0133 Loki と同 stack) |
| **Arize Phoenix** | OTel-native spans、branching paths capture、ELv2 license | AgentRunEvent timeline visualization、provider event branching |
| **AgentOps** | Time-travel debugging、replay agent runs at any point | ContextSnapshot 10 列 replay engine 化 (P1 候補) |
| **Latitude** | Full causal session trace、step relationships、agent rollout debugger | AgentRun parent_run_id graph、multi-trace workflow view |
| **Helicone** | Drop-in proxy、Apache-2.0、最 simple install | Provider Adapter proxy 化候補 (P1) |

### 1.2 業界 共通 best practice (2026)

- **Keyboard-first philosophy**: 全 action を keyboard で完結 (mouse 不要)
- **Command palette (Cmd+K)**: search + action 統合
- **Human-in-the-loop checkpoints**: confidence score / risk threshold で trigger
- **Sandboxed execution**: Docker / Firecracker isolated environment
- **Time-travel debugging**: agent run の任意 point から replay
- **Session replay**: transcript view + signals + SQL over traces
- **OTel-native instrumentation**: span による causal trace
- **Step-level cost attribution**: provider call ごと token / $ 分解
- **Multi-trace workflow view**: parent / child run graph
- **Killswitch / rollback / RBAC**: 運用時 safety net

## 3. 提案機能 (優先度別 + 不変条件 trace)

### 3.1 P0 must_ship (5 機能、本 PR scope 内候補、軽量 + 大 UX 改善)

各機能は SP-024 候補として Sprint Pack 化、または SP-009 must_ship 拡張で扱う。

#### P-HIGH-01: Command Palette (Cmd+K) skeleton + 主要 5 action

- 操作: `Cmd+K` で modal、search input + action list
- action 5 種 (P0): create ticket / open Approval Inbox / open Agent Runs / open Audit Log / open Settings
- 実装: React component (Client Component、Server Action 経由で navigation)
- 不変条件 trace: #1 AI 直結禁止 (command palette は AI 出力 dispatcher ではない、純 UI navigation のみ) / #14 doc-only gate (本提案は doc、実装は別 PR)
- gap 解消: Linear pattern を最小限導入

#### P-HIGH-02: Keyboard shortcut minimal set

- shortcut 8 個 (P0): j/k navigation、Cmd+Enter approve、Esc cancel、g + i (go inbox)、g + r (go runs)、g + a (go audit)、g + t (go tickets)、? (help modal)
- 実装: Global keyboard event listener (Client Component)
- 不変条件 trace: #10 Self-approval 禁止 (Cmd+Enter approve は actor が requester でないこと server-side で再 verify) / #13 server-owned (shortcut は client-side hint、server-side validation 必須)

#### P-HIGH-03: Agent Run Timeline visualization (graph view)

- 現状: 16 状態の list 表示
- 改善: SVG graph で 16 状態 + blocked_reason 3 種 + 主要 transition を visualize
- AgentRunEvent timeline を chronological node 表示
- 実装: React component (D3 or visx)、Server Component で AgentRunEvent fetch
- 不変条件 trace: #6 AgentRun 16 状態 (enum 不変、graph は表示形態のみ変更) / #8 ContextSnapshot 10 列 (各 node に snapshot metadata link、内容 immutable)

#### P-HIGH-04: Approval Inbox bulk action

- 操作: multi-select (checkbox + Shift+click range)、bulk approve / bulk reject
- 制約: 同一 actor が requester の approval は bulk から除外 (self-approval invariant)
- 4 整合 binding (artifact_hash / policy_version / provider_request_fingerprint / action_class) を bulk でも各 row ごと verify
- 実装: Server Action で bulk processing、各 approval は独立に validate
- 不変条件 trace: #10 Self-approval 禁止 / #13 server-owned (bulk でも 4 整合は server で再計算) / #1 AI 直結禁止 (AI は提案、人間 bulk approve のみ)

#### P-HIGH-05: Universal search (artifact 横断)

- search target: Ticket / AgentRun / Approval / Audit Event / Sprint Pack metadata
- tenant_id / project_id boundary 厳守 (cross-tenant 検索禁止)
- raw secret / capability token 値 除外 (audit event は redacted のみ)
- 実装: Postgres tsvector + GIN (P0.1 で pgvector 検討、本 P0 は tsvector のみ)
- 不変条件 trace: #9 PostgreSQL invariant (tenant_id / project_id を全 query に必須) / #5 SecretBroker (raw secret 検索結果に含めない) / #14 doc-only gate (P0.1+ で実装、本提案は spec)
