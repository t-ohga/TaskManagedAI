## Cross-Cutting Lessons

- Sample は採用候補の宝庫だが、TaskManagedAI では必ず `ProviderAdapter`、`Tool Gateway`、`Policy`、`Approval`、`SecretBroker`、`AuditEvent` を通して再設計する。
- Cloud managed search / RAG / Agent Engine は便利でも、P0 の source of truth にはしない。
- UI に出す価値が高いのは「モデルの答え」ではなく、`request fingerprint`、`tool trajectory`、`retrieval query`、`ranking policy`、`citation trace`、`review verdict`。
- source evidence は `raw/evidence_index.md` の source-root relative path と line/cell anchor で再検証する。短い省略 path は使用しない。
- provider-managed execution / browser / fetch / batch / live / model routing は、同じ `gemini` name でも risk と data flow が異なる。feature 単位で provider compliance と gate を切らない限り、P0 実装に混ぜない。
- Sprint Pack に流すときは global P0 backlog ID を正本にし、Sprint Pack-local `BL-*` の衝突を translation table か namespace で解消してから実装する。
